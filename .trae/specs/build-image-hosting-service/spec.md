# 圖片託管服務 (Image Hosting Service) Spec

## 為什麼 (Why)
現有的圖床服務無法完全滿足高自由度的客製化需求。因此，需要從零打造一個功能強大且具有現代化管理介面的專屬圖床系統。該系統需具備高度靈活的存儲策略配置、強大的圖片管理功能、完善的角色組權限控制，並能與現有的基礎設施（PostgreSQL, Redis）及第三方服務（Cloudflare R2, 七牛雲, Waline 評論）無縫整合。

## 變更內容 (What Changes)
- 構建全新的前端圖片管理與展示介面，支援瀑布流、右鍵選單操作（重命名、複製連結）、多選等進階管理功能。
- 開發支援多種上傳方式（多圖、拖拽、貼上）的 Web UI 與 API 接口。
- 實作後端 API，支援多種存儲策略（如 Cloudflare R2、七牛雲）並能動態切換。
- 整合 PostgreSQL 作為核心資料庫，存儲用戶、角色、圖片元數據；Redis 作為快取與上傳頻率限制。
- 實作高自由度的角色組配置：為不同角色組分配不同的存儲策略、上傳路徑與命名規則、頻率限制及圖片審核機制。
- 開發強大的圖片處理模組，支援圖片/文字浮水印、平鋪效果、自定義 X/Y 軸偏移量及旋轉角度。
- 實作對接 Waline 評論系統的 API，方便直接在評論中使用上傳的圖片。
- 提供對外的 API 接口，支援第三方應用的圖片上傳、圖片管理及相簿管理。
- 構建圖片廣場（公開畫廊）功能，用於展示公開圖片。
- 實作線上增量更新與跨版本更新機制，確保系統易於維護與升級。

## 影響範圍 (Impact)
- Affected specs: 全新的圖床系統架構與功能模組。
- Affected code: 全新的後端 API 服務與前端管理面板（從零開發）。

## 新增需求 (ADDED Requirements)
### Requirement: 圖片上傳與強大管理功能
系統應提供多種上傳方式（多圖、拖拽、貼上），並具備直觀且功能豐富的圖片管理介面。
#### Scenario: 瀑布流管理與進階操作
- **當** 用戶進入圖片管理頁面
- **則** 系統以瀑布流展示圖片，並支援透過右鍵選單或多選進行重命名、一鍵複製連結等操作。

### Requirement: 極高自由度的存儲與角色策略
系統應允許配置多個存儲策略，並可靈活綁定至不同的角色組。
#### Scenario: 角色組自定義限制與策略
- **當** 特定角色組的用戶上傳圖片時
- **則** 系統根據該組的設定（上傳文件/文件夾路徑命名規則、上傳頻率限制、圖片審核）處理上傳，並存儲至指定的動態策略（R2 或七牛雲）。

### Requirement: 圖片處理與浮水印
系統應支援在上傳時動態添加高度自定義的浮水印。
#### Scenario: 添加文字與圖片浮水印
- **當** 用戶設定了浮水印規則（圖片/文字、平鋪、X/Y軸偏移、旋轉角度）
- **則** 系統在上傳或處理圖片時自動應用這些視覺處理。

### Requirement: 系統整合與開放 API
系統需對接 Waline 評論，並提供外部管理 API 及圖片廣場。
#### Scenario: 第三方調用與 Waline 對接
- **當** 外部應用透過 API 提交圖片，或從 Waline 發起上傳請求
- **則** 系統驗證權限後，按對應策略存儲並返回圖片連結；同時，公開圖片將展示於「圖片廣場」。
